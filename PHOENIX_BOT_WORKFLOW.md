## 总控模式消息格式

### 标准格式

```
@Bot 名 [指令类型 | 请求 ID| 小小谦] 任务内容
```

### 指令类型

| 类型 | 用途 | 示例 |
|------|------|------|
| `ASK` | 询问/请求 | `@场控 [ASK|R001| 小小谦] 现在直播间多少人？` |
| `DO` | 执行任务 | `@剪辑 [DO|R002| 小小谦] 把刚才的高光剪成 60 秒视频` |
| `CONFIRM` | 确认收到 | `@小小谦 [CONFIRM|R001| 场控] 已收到` |
| `REPORT` | 进展汇报 | `@小小谦 [REPORT|R001| 场控] 当前直播间 500 人` |
| `DONE` | 任务完成 | `@小小谦 [DONE|R002| 剪辑] 已生成 3 个切片` |
| `FAIL` | 任务失败 | `@小小谦 [FAIL|R002| 剪辑] 无法完成，源视频损坏` |

## Bot 间协作规则

### ✅ 允许的行为

1. **跨 Bot 信息共享**（非正式任务）
   ```
   [场控] @剪辑 刚才那段直播数据很好，建议切片
   [剪辑] 收到，等场控给具体时间点
   ```

2. **紧急情况下的直接沟通**
   ```
   [场控] @编导 主播临时需要互动话题，在线人数掉了
   [编导] 马上给！话题：你最喜欢的一首歌
   ```

3. **任务交接**
   ```
   [剪辑] @渠道 3 个切片视频已交付
   [渠道] 收到，开始分发到各平台
   ```

### ❌ 禁止的行为

1. **跨 Bot 派单**
   ```
   ❌ [运营] @剪辑 做个视频
   ✅  [运营] @小小谦 建议安排@剪辑 做个视频
   ```

2. **不确认接收者**
   ```
   ❌ "把视频剪一下"
   ✅ "@剪辑 [DO|R003| 小小谦] 把视频剪成 60 秒"
   ```

3. **没有请求 ID**
   ```
   ❌ "@场控 查看数据"
   ✅ "@场控 [ASK|R004| 小小谦] 查看数据"
   ```

## 各 Bot 专属能力

### 场控（Live Controller）

**专属 API**:
- `GET /api/agent/dashboard` - 实时数据
- `POST /api/agent/memory` - 写入记忆

**能做的事**:
- 读取直播间在线人数、弹幕密度
- 写入关键节点到 `memory/YYYY-MM-DD.md`
- 提炼流量密码到 `MEMORY.md`

**不能做的事**:
- 不能直接命令@剪辑 切片
- 不能直接命令@运营 分析

---

### 运营（Data Analyst）

**专属 API**:
- `GET /api/agent/trends` - 流量趋势
- `GET /api/agent/comparison` - 竞品对比

**能做的事**:
- 读取完整直播数据仪表盘
- 生成日报/周报
- 用户画像分析

**不能做的事**:
- 不能直接命令@场控 监控
- 不能直接命令@编导 改脚本

---

### 编导（Content Creator）

**专属 API**:
- `GET /api/agent/trending` - 热点话题
- `POST /api/agent/script` - 提交脚本

**能做的事**:
- 获取热点趋势
- 创作直播脚本
- 设计互动环节

**不能做的事**:
- 不能直接命令@剪辑 制作
- 不能直接命令@渠道 分发

---

### 剪辑（Video Editor）

**专属 API**:
- `POST /api/agent/video/cut` - 直播切片
- `POST /api/agent/video/convert` - 格式转换

**能做的事**:
- 使用 FFmpeg 进行视频处理
- 制作 30-60 秒高光切片
- 制作 3-5 分钟完整版

**不能做的事**:
- 不能直接命令@渠道 发布
- 不能直接命令@美工 设计

---

### 渠道（Distributor）

**专属 API**:
- `POST /api/agent/distribute` - 内容分发
- `GET /api/agent/channel-data` - 渠道数据

**能做的事**:
- 多平台内容分发
- 渠道效果分析
- 最佳时间发布

**不能做的事**:
- 不能直接命令@剪辑 制作
- 不能直接命令@编导 创作

---

### 客服（Fan Manager）

**专属 API**:
- `GET /api/agent/fan-list` - 粉丝列表
- `POST /api/agent/fan-tag` - 粉丝标签

**能做的事**:
- 粉丝问题解答
- 私域社群运营
- 问题收集反馈

**不能做的事**:
- 不能直接命令@场控 监控
- 不能直接命令@运营 分析

---

### 美工（Designer）

**专属 API**:
- `GET /api/agent/templates` - 设计模板
- `GET /api/agent/brand-assets` - 品牌资产

**能做的事**:
- 直播封面设计
- 品牌 VI 设计
- OBS 直播间包装

**不能做的事**:
- 不能直接命令@渠道 发布
- 不能直接命令@编导 写文案

---

### 小小谦（Master Controller）

**专属能力**:
- 唯一可以调度所有 Bot 的总控
- 跨 Bot 协调
- 任务优先级管理

**责任**:
- 接收所有 Bot 的 `CONFIRM/REPORT/DONE/FAIL`
- 分配任务给对应 Bot
- 处理 Bot 间的依赖关系

## 示例：完整任务链路

```
1. [运营] 发现昨晚直播 B 站数据很好
   → @小小谦 [REPORT|R010| 运营] B 站涨粉 500 人，建议加大 B 站投入

2. [小小谦] 分析建议
   → @编导 [DO|R011| 小小谦] 策划 B 站专属内容
   → @剪辑 [DO|R012| 小小谦] 制作横版 16:9 视频适配 B 站
   → @渠道 [DO|R013| 小小谦] 研究 B 站运营策略

3. [编导] 输出方案
   → @小小谦 [DONE|R011| 编导] B 站专属脚本：3 分钟深度解析

4. [剪辑] 交付视频
   → @小小谦 [DONE|R012| 剪辑] 横版视频 x3 已交付

5. [渠道] 发布 + 监控
   → @小小谦 [DONE|R013| 渠道] B 站视频已发布，24 小时后看数据

6. [运营] 数据验证
   → @小小谦 [REPORT|R014| 运营] B 站视频 24h 数据：播放 10w，涨粉 200
```

## 记忆系统

### 场控写入的记忆

**开播记录** (`memory/YYYY-MM-DD.md`):
```markdown
## 2026-04-11 开播记录

- 开播时间：19:00
- 初始在线：50 人
- 封面标题：《谦歌行 277 - 经典老歌回忆杀》
```

**关键节点** (`memory/YYYY-MM-DD.md`):
```markdown
### 高光时刻 20:15

- 在线峰值：1200 人
- 弹幕密度：500 条/分钟
- 事件：主播演唱《突然的自我》
```

**下播总结** (`memory/YYYY-MM-DD.md`):
```markdown
### 下播总结

- 累计观看：5000 人
- 平均在线：800 人
- 峰值在线：1200 人 (20:15)
- 新增粉丝：300 人
- 礼物收入：2000 元
```

## 流量密码

1. 老歌回忆杀 → 高互动
