# 共享上下文系统设计

## 问题
现在每个 Bot 的记忆是独立的，协作时会出现：
- 编导不知道运营做了什么决定
- 剪辑不知道场控的安排
- 信息孤岛，协作效率低

## 解决方案：共享上下文

### 架构
```
共享记忆层 (Shared Memory Layer)
├── shared_memory/MEMORY.md      ← 团队共享记忆
├── shared_memory/USER.md        ← 用户偏好
├── shared_memory/projects/      ← 项目上下文
└── shared_memory/logs/          ← 每日共享日志

各 Bot 工作空间 (Workspace)
├── workspaces/编导/memory/      ← 编导私有记忆
├── workspaces/运营/memory/      ← 运营私有记忆
└── ...
```

### 写入规则

**自动写入共享日志的情况：**
1. Bot 收到用户@并回复
2. Bot 之间互相@协作
3. 任务完成/失败汇报
4. 重要决策/结论

**不写入的情况：**
1. 简单确认（"收到"，"在的"）
2. Bot 间协调细节
3. 重复信息

### 读取规则

**每个 Bot 启动时自动加载：**
1. 自己的私有记忆
2. 共享上下文（只读）

**系统提示注入顺序：**
```
SOUL.md → IDENTITY.md → AGENTS.md → TOOLS.md → 
SHARED_CONTEXT (只读) → PRIVATE_MEMORY → USER.md
```

### 实现步骤

1. 创建 `shared_memory/` 目录结构
2. 修改 `phoenix_gateway.py` 注入共享上下文
3. 修改 `discord_bot.py` 自动记录共享日志
4. 创建共享上下文查看工具

## 示例日志格式

```markdown
## 2026-04-10 共享日志

### [13:15] 小小谦 → 运营
- **内容**: 谦总问今晚直播准备得怎么样
- **运营回复**: 音乐和灯光已准备好，背景板 7 点到位

### [13:20] 运营 → 全体
- **内容**: 今晚直播流程确认
- **详情**: 
  1. 19:30 开场暖场
  2. 20:00 正式开始
  3. 21:30 互动环节
  4. 22:00 结束

### [13:25] 小小谦 → 谦总
- **内容**: 汇报直播准备情况
- **结论**: 一切就绪
```

## 注意事项

1. 共享日志要简洁，不要流水账
2. 敏感信息（如密码、密钥）不共享
3. 每个 Bot 还是可以有自己的私有笔记
